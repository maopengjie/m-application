# Decidely V1.1 功能拆解与开发任务清单

## 1. 文档目的

这份文档用于将 [V1.1 需求清单](/Users/maopengjie/Documents/m-application/docs/v1.1-requirements.md) 进一步拆解为可执行研发任务，方便产品、设计、前端、后端和测试协同推进。

适用范围：

- V1.1 可信度优化版
- 周级排期
- 任务分工
- 联调和验收

## 2. V1.1 目标

V1.1 的核心目标是把 V1.0 从“能用”升级到“可信、好用、可验证”。

核心结果包括：

- 决策建议模块更可信
- 商品详情页更有解释力
- 提醒模块更有回访价值
- 产品具备基础埋点和指标验证能力

## 3. 任务优先级定义

- `P0`：版本核心项，直接影响 V1.1 目标达成
- `P1`：重要增强项，不阻塞版本上线，但影响体验和验证效果
- `P2`：可延后项，本期不进入主计划

## 4. V1.1 模块拆解

## 5. 周级开发排期建议

### 第 1 周

- 复盘 V1.0 数据和问题
- 冻结 V1.1 范围
- 确认建议卡结构和埋点方案
- 明确详情页增强项字段需求

### 第 2 周

- 完成建议卡后端字段补充
- 完成建议卡前端改造
- 完成价格位置说明和店铺标签增强
- 完成到手价表达优化

### 第 3 周

- 完成提醒状态增强
- 完成替代商品推荐轻量版
- 完成埋点接入

### 第 4 周

- 完成联调和回归测试
- 输出基础指标看板
- 形成 V1.1 可验收版本

## 6. 分工建议

### 产品

- 定义建议卡结构和文案规则
- 定义价格位置、推荐方案、风险说明表达
- 明确埋点口径和指标定义

### 前端

- 实现新版建议卡
- 增强详情页解释层
- 改造提醒状态展示
- 接入关键埋点

### 后端

- 补充建议解释字段
- 补充价格位置相关字段
- 校验风险和提醒状态逻辑
- 提供或复用替代商品数据接口

### 测试

- 建议卡多状态测试
- 详情页说明与 SKU 切换测试
- 提醒状态与回流测试
- 埋点触发测试

## 7. 联调清单

- 建议卡字段是否齐全且与文案规则一致
- SKU 切换后建议、价格、优惠、风险是否同步更新
- 到手价与优惠说明是否一致
- 风险标签和店铺标签是否准确
- 提醒状态是否正确显示并可回流详情
- 埋点是否在正确时机触发

## 8. 验收清单

### 功能验收

- 建议卡完整展示结论、证据、风险和动作
- 详情页可解释当前价格位置
- 用户能区分推荐方案和最低价方案
- 提醒页状态表达更清晰
- 替代商品在建议等待场景可正常展示

### 数据验收

- 首页搜索埋点可用
- 搜索结果点击详情埋点可用
- 去购买埋点可用
- 提醒创建埋点可用

### 体验验收

- 用户能够快速理解为什么建议买或等
- 详情页解释力增强，不再只是信息堆叠
- 提醒页更像回访入口，而不是静态列表

## 9. 风险与阻塞项

- 后端若无法提供足够解释字段，前端只能做表层优化
- 替代商品推荐质量不足时，可能影响用户信任
- 埋点方案若口径不统一，后续指标不可用
- 若详情页继续增加模块但不收口层级，反而会降低可读性
