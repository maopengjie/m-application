# 从 0 到 1 的整套 UI 准备清单 + 页面规划模板

> 适用项目：`M-Application` 当前阶段的智能购物决策平台 MVP  
> 当前技术栈：`Vue 3 + Element Plus + Vben Admin + FastAPI`

## 1. 先基于当前项目，明确“这套 UI 是什么”

### 1.1 产品定位

- 产品名称：智能购物决策平台 / Decidely
- 产品目标：帮助用户解决“买不买、去哪买、什么时候买、有没有风险”
- 当前 MVP 主线：搜索商品、查看详情、比较平台价格、观察历史价格、获取 AI 决策、设置降价提醒

### 1.2 当前项目建议采用的用户角色

当前代码和 PRD 显示，MVP 更偏 C 端决策工具，而不是多角色后台系统。建议先拆成两层：

#### MVP 必备角色

- 游客/普通用户
  - 搜索商品
  - 查看商品详情
  - 查看比价、历史价格、优惠、风险、决策建议
  - 设置与管理降价提醒

#### 第二阶段扩展角色

- 登录用户
  - 持久化提醒、收藏、浏览记录、个性化推荐
- 运营管理员
  - 管理商品数据质量、监控采集任务、处理风险规则、维护推荐策略
- 数据分析/业务管理员
  - 查看搜索热度、提醒转化、平台价格波动、规则命中效果

### 1.3 当前项目的主要业务场景

- 查询：搜索商品、查看商品详情、查看提醒列表
- 分析：比价、历史价格走势、优惠拆解、风险判断、AI 决策建议
- 录入：设置目标价提醒
- 管理：取消提醒、后续扩展个人偏好设置
- 统计：后续可扩展搜索转化、提醒命中、平台价格趋势统计

### 1.4 当前项目的关键页面

根据现有代码与路由，当前核心页面为：

- 登录页
- 首页 / 搜索入口
- 搜索结果页
- 商品详情页
- 降价提醒页
- 个人设置页
- 仪表盘页（Analytics / Workspace）

### 1.5 当前项目的重点流程

当前最核心流程建议定义为：

1. 首页输入关键词
2. 进入搜索结果页
3. 点击商品进入详情页
4. 查看价格趋势 / 同款比价 / 优惠 / 风险 / AI 决策
5. 设置降价提醒
6. 在提醒页统一管理提醒

可以沉淀成一句产品流程：

`搜索商品 -> 对比价格 -> 理解决策 -> 设置提醒 -> 等待触达`

## 4. 基于当前项目的交互规则清单

### 4.1 全局交互规则

- 搜索提交：
  - 关键词为空时不发请求
  - 按回车与点击按钮等效
- 页面加载：
  - 首次进入有 loading
  - 请求失败有 message 或错误态
- 空状态：
  - 搜索无结果时展示空状态和重试建议
  - 提醒为空时引导回搜索页

### 4.2 搜索结果页交互

- 当 URL 带 `q` 参数时自动触发搜索
- 搜索新关键词时同步更新路由 query
- 点击列表项进入详情页
- 后续扩展时，筛选、分页、排序建议统一回写 URL，便于分享与刷新恢复

### 4.3 商品详情页交互

- 点击“降价提醒”打开弹窗
- 提交提醒成功后给出成功提示
- 获取详情失败时展示空态并支持返回
- “去购买”建议后续补齐真实跳转链接

### 4.4 删除和危险操作

- 删除提醒必须二次确认
- 二次确认文案要明确说明影响范围

### 4.5 建议补齐的状态规范

- Loading：骨架屏优先于整页转圈
- Empty：给出下一步动作
- Error：说明失败原因 + 重试入口
- Disabled：说明为什么不可用
- No Permission：说明缺少权限和联系人/申请路径

### 4.6 权限建议

当前 MVP 可简化为：

- 游客：可搜索、可查看详情
- 登录用户：可创建和管理提醒
- 管理员：后续可进入数据后台、规则配置、运营页面

### 6.3 建议拆分规则

- 输入输出稳定
- 在多个页面复用

#### 业务组件

- 带业务语义
- 面向某个业务模块
- 可组合多个通用组件

例如：

- 通用组件：`BaseCard`、`BaseFilterBar`
- 业务组件：`DecisionCard`、`PriceTrendChart`

## 7. 数据与状态准备清单

### 7.1 每个页面都要先补齐的数据问题

- 列表数据从哪个接口来
- 详情数据从哪个接口来
- 表单提交到哪个接口
- 哪些字段是枚举/字典
- 哪些字段需要异步加载
- 哪些字段需要默认值
- 哪些字段会影响其他字段联动

### 7.2 当前项目已识别接口

| 页面/功能 | 接口 | 用途 |
| :--- | :--- | :--- |
| 搜索结果 | `GET /search` | 搜索商品 |
| 商品详情 | `GET /products/{id}` | 获取商品详情 |
| 创建提醒 | `POST /alerts` | 创建降价提醒 |
| 提醒列表 | `GET /alerts` | 获取提醒列表 |
| 删除提醒 | `DELETE /alerts/{id}` | 取消提醒 |

### 7.3 当前项目的关键状态

- 商品搜索：
  - 未搜索
  - 搜索中
  - 有结果
  - 无结果
  - 搜索失败
- 商品详情：
  - 加载中
  - 加载成功
  - 加载失败
- 决策结果：
  - `BUY`
  - `WAIT`
  - `AVOID`
- 风险等级：
  - `LOW`
  - `MEDIUM`
  - `HIGH`
- 提醒状态：
  - 待触发
  - 已触发
  - 已取消
