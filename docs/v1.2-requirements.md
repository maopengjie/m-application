# Decidely V1.2 需求清单

## 1. 版本目标

V1.2 的核心目标不是继续增强“单次决策”，而是验证这个产品是否具备持续留存价值。

一句话定义：

`让用户不仅做一次决策，还愿意持续回来查看。`

V1.2 聚焦三件事：

- 把提醒做成真正的回访入口
- 增加关注商品与商品动态能力
- 建立留存相关指标验证体系

如果说：

- V1.0 = 能用
- V1.1 = 可信

那么：

- V1.2 = 会回来

## 2. 版本定位

### 2.1 V1.2 要解决的问题

V1.2 主要回答以下问题：

- 用户在设置提醒后是否会因为价格变化而回访
- 用户是否愿意持续关注一组商品，而不是只看一次详情页
- 产品是否可以从“单次决策工具”升级为“持续关注工具”

### 2.2 V1.2 核心结果

- 提醒不再只是状态列表，而成为回访入口
- 用户可以管理并关注一组商品
- 产品拥有可观测的留存相关指标

## 3. 核心需求

### 3.1 提醒回访链路增强

#### 目标

让提醒不只是状态变化，而是能带来再次决策。

#### 需求项

- 已触发提醒支持一键返回商品详情
- 提醒卡片展示触发原因，例如“已低于目标价”
- 提醒列表支持突出“建议立即查看”的商品
- 在提醒触发后展示新的决策状态
- 支持从提醒页直接进入去购买或看替代品

#### 验收标准

- 已触发提醒不再只是静态标记
- 用户能从提醒列表快速回流到详情页
- 提醒页具备行动承接能力

### 3.2 关注商品机制

#### 目标

让用户不只关注“价格提醒”，也能持续关注商品变化。

#### 需求项

- 支持收藏或关注商品
- 关注商品可进入独立列表
- 列表展示当前价、价格变化、风险状态、是否接近低点
- 支持从关注列表进入详情页
- 支持取消关注

#### 验收标准

- 用户可管理自己关注的商品
- 关注列表不只是收藏夹，而是动态信息面板
- 用户有理由反复回看

### 3.3 商品异动聚合页

#### 目标

把零散提醒和关注商品变成一个统一的信息入口。

#### 需求项

- 聚合展示关注商品中的关键变化
- 支持展示：
  - 最近降价
  - 接近目标价
  - 已到历史低位
  - 风险变化
- 支持按“最值得看”排序
- 支持从聚合页进入详情页

#### 验收标准

- 用户无需逐个点开商品也能知道哪些值得关注
- 页面具备“今日该看什么”的引导感

### 3.4 详情页二次访问优化

#### 目标

让用户回访详情页时，不需要重新理解上下文。

#### 需求项

- 标记用户是否已关注或已设置提醒
- 如果商品有新变化，详情页首屏展示变化摘要
- 增加“上次看后发生了什么”提示
- 对回访用户优先展示最近变化和建议更新

#### 验收标准

- 回访用户能快速进入状态
- 详情页对二次访问用户更有承接力

### 3.5 留存指标体系

#### 目标

验证产品是否真的有持续使用价值。

#### 核心指标

- 提醒触发回访率
- 7 日复访率
- 二次详情访问率
- 关注商品数
- 关注商品详情点击率
- 提醒转购买率
- 聚合页访问率

#### 验收标准

- 能清晰看出提醒、关注、聚合页对留存的贡献
- 能支持 V2 是否继续强化留存能力的判断

## 4. 优先级建议

### 4.1 P0

- 提醒回访链路增强
- 关注商品机制
- 留存指标埋点补齐

### 4.2 P1

- 商品异动聚合页
- 详情页二次访问优化
- 关注商品动态状态表达

### 4.3 P2

- 更智能的异动推荐排序
- 个性化关注建议
- 周报或月报类内容化触达

## 5. 排期建议

### 第 1 周

- 复盘 V1.1 指标
- 冻结 V1.2 范围
- 定义关注商品与提醒回访规则
- 明确留存埋点方案

### 第 2 周

- 做提醒回访链路增强
- 做关注商品基础能力
- 接入核心留存埋点

### 第 3 周

- 做商品异动聚合页
- 做详情页二次访问优化
- 联调关注与提醒数据流

### 第 4 周

- 回归测试
- 验证关键留存指标
- 决定下一阶段是强化增长还是做商业化验证

## 6. 与 V1.1 的关系

V1.2 不应重新定义产品价值，而应在 V1.1 已建立的可信决策基础上，继续强化用户的持续访问理由。

原则如下：

- 不破坏已有决策链路
- 优先强化提醒和关注的回访价值
- 所有新页面和新模块都要服务于留存验证
- 避免加入与当前版本目标无关的大功能

## 7. 风险与注意事项

- 如果关注商品列表只是“收藏夹”，用户不会形成持续使用习惯
- 如果提醒触发后没有清晰动作，回访价值会被削弱
- 如果异动聚合页没有排序逻辑，信息会显得杂乱
- 如果留存埋点口径不统一，V1.2 复盘将缺乏结论性

## 8. 结论

V1.2 最重要的不是再增加新的“决策功能”，而是把一次性的购买前查询，变成持续性的商品关注关系。

如果说 V1.1 的目标是“让用户愿意依据建议行动”，那么 V1.2 的目标就是“让用户愿意反复回来，持续依据这些变化做决定”。
