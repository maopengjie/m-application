# Decidely V2 个人版功能拆解与开发任务清单

## 1. 文档目的

这份文档用于将 [V2 个人版需求清单](/Users/maopengjie/Documents/m-application/docs/v2-personal-requirements.md) 进一步拆解为可执行研发任务，方便产品、设计、前端、后端和测试协同推进。

适用范围：

- V2 个人版
- 周级排期
- 任务分工
- 联调和验收

## 2. V2 目标

V2 的核心目标是把产品从“可回访的决策工具”升级为“可长期使用的个人购物决策工作台”。

核心结果包括：

- Watchlist 成为个人商品决策中心
- 商品动态工作台成为日常查看入口
- 决策建议更像私人购物顾问
- 大促场景和轻分享提升实用性

## 3. 任务优先级定义

- `P0`：版本核心项，直接影响 V2 长期使用价值
- `P1`：重要增强项，不阻塞版本上线，但影响体验完整度
- `P2`：可延后项，本期不进入主计划

## 6. 分工建议

### 产品

- 定义 Watchlist 字段、动态页信息结构和建议话术规则
- 明确大促模式和分享卡片范围
- 明确 V2 验证指标

### 前端

- 实现 Watchlist 增强、动态工作台、新版建议卡
- 实现大促筛选和轻分享
- 接入关键埋点

### 后端

- 扩展 Watchlist 数据模型和接口
- 提供动态聚合接口与建议增强字段
- 提供大促模式所需状态字段
- 支撑指标看板输出

### 测试

- Watchlist 增删改查测试
- 动态工作台聚合正确性测试
- 建议卡多状态测试
- 分享和埋点触发测试

## 7. 联调清单

- Watchlist 字段是否正确保存和回显
- 动态工作台是否准确展示关键变化
- 详情页建议是否展示更自然的话术和观察点
- 大促模式标签和筛选是否正确
- 分享卡片内容是否准确
- 埋点是否在正确节点触发

## 8. 验收清单

### 功能验收

- 用户可对 Watchlist 商品进行分组、备注、设置目标价
- 动态工作台可展示值得优先查看的商品变化
- 建议卡更像顾问建议而不是模板结果
- 大促场景下可快速筛出重点商品
- 详情页可生成轻量分享卡片

### 数据验收

- Watchlist 使用相关埋点可用
- 动态页访问和点击埋点可用
- 分享行为埋点可用
- 可查看 V2 使用验证数据

### 体验验收

- Watchlist 像个人商品工作台而不是收藏夹
- 动态页像“今天该看什么”的入口
- 建议和分享都更贴近日常使用场景

## 9. 风险与阻塞项

- 如果 Watchlist 缺少足够上下文信息，会沦为普通关注列表
- 如果动态工作台排序不好，页面会变成信息噪音
- 如果建议升级只停留在文案层，用户感知提升有限
- 如果大促模式过重，可能偏离“个人工具”定位
