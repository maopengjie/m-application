# Decidely V2 个人版需求清单

## 1. 版本目标

V2 不再以商业化为目标，而是把产品打造成一个适合你和朋友长期使用的 `个人购物决策工作台`。

一句话定义：

`买贵东西之前先查一下，平时把想买的东西放进去慢慢盯。`

V2 聚焦三件事：

- 提升个人长期使用价值
- 强化 watchlist 和商品动态能力
- 把产品从“查询工具”升级为“日常决策工作台”

如果说：

- V1.0 = 能用
- V1.1 = 可信
- V1.2 = 会回来

那么：

- V2 = 愿意长期用

## 2. 产品定位

V2 的定位不是导购平台，也不是电商站，而是：

`个人购物决策助手`

适用场景：

- 你想买一台手机，但不急着立刻买
- 你和朋友都在盯一些贵价商品
- 你想知道最近哪些商品值得重新看一眼
- 你想在大促前判断“现在别买，等活动”

## 3. 核心需求

### 3.1 Watchlist 增强

#### 目标

把“关注商品”变成个人决策中心。

#### 需求项

- 支持关注商品分组
- 支持填写关注原因
- 支持填写目标价或理想价
- 支持备注，例如“给朋友推荐”“等 618”“准备换机”
- 展示当前价、价格变化、风险状态、接近低点程度
- 支持排序和筛选

#### 验收标准

- Watchlist 不只是收藏夹，而是带意图的个人商品清单
- 用户能快速知道哪些商品值得继续盯

### 3.2 商品动态工作台

#### 目标

让用户每天打开一次就知道最近有什么值得看。

#### 需求项

- 聚合 watchlist 商品动态
- 展示：
  - 最近降价
  - 接近目标价
  - 已到历史低位
  - 风险变化
  - 推荐重新查看
- 支持按“最值得看”排序
- 支持按分组或场景筛选

#### 验收标准

- 页面具备“今天先看什么”的价值
- 用户不需要逐个点开商品才能知道变化

### 3.3 决策建议升级

#### 目标

让建议更像私人购物顾问，而不是系统提示。

#### 需求项

- 增加更自然的建议语气
- 支持给出“建议等多久”或“建议观察什么”
- 在建议等待时给出更具体的等待理由
- 提供替代商品建议
- 提供“适合现在买的人群”提示

#### 验收标准

- 建议更像给人看的建议，而不是规则输出
- 用户能直接知道下一步做什么

### 3.4 大促场景模式

#### 目标

提升 618、双11 等大促时期的实用性。

#### 需求项

- 标记 watchlist 中适合大促关注的商品
- 增加“大促前不建议买”提示
- 大促期间展示“最值得立即下单”商品
- 支持快速筛选“本次活动最值得买”
- 对价格预热和历史活动低点做简单对比

#### 验收标准

- 大促前后用户会明显更愿意打开产品
- watchlist 和动态页在活动期价值提升明显

### 3.5 朋友间轻分享

#### 目标

适合小范围朋友使用，而不是社交平台。

#### 需求项

- 商品详情可生成简洁分享卡片
- 支持分享 watchlist 中单个商品
- 支持附带一句推荐理由
- 支持“我为什么在等这个价格”的说明

#### 验收标准

- 分享是轻量、实用的，不引入复杂社交关系
- 适合朋友之间讨论买不买

## 4. 优先级建议

### 4.1 P0

- Watchlist 增强
- 商品动态工作台
- 决策建议升级

### 4.2 P1

- 大促场景模式
- 朋友间轻分享
- Watchlist 分组和备注优化

### 4.3 P2

- 更复杂的个性化建议
- 自动识别更适合的购买窗口
- 更细的多用户协作能力

## 5. 排期建议

### 第 1 周

- 复盘 V1.2 使用情况
- 冻结 V2 个人版范围
- 明确 watchlist 字段设计
- 明确动态工作台展示规则

### 第 2 周

- 做 watchlist 增强
- 做关注分组、目标价、备注能力
- 做详情页关注信息增强

### 第 3 周

- 做商品动态工作台
- 做动态排序和筛选
- 做建议语气和建议结构升级

### 第 4 周

- 做大促模式基础支持
- 做轻量分享能力
- 联调和回归测试

## 6. 与 V1.2 的关系

V2 个人版不是推翻 V1.2，而是在已有的提醒、回访和留存基础上，把产品进一步升级为长期个人使用工具。

原则如下：

- 不破坏已有搜索、详情、提醒主链路
- 所有新增能力都要围绕个人使用效率和长期价值
- 不引入与当前定位无关的商业化逻辑
- 控制功能复杂度，避免产品从“个人工具”膨胀成“泛平台”

## 7. 风险与注意事项

- 如果 watchlist 没有足够动态信息，用户会把它当成普通收藏夹
- 如果动态工作台只是信息堆叠，用户不会形成每日查看习惯
- 如果大促模式没有足够实用价值，会成为低频功能
- 如果分享能力过重，容易偏离“个人和朋友使用”的定位

## 8. 结论

V2 个人版最重要的，不是做成一个更大的平台，而是让它成为你和朋友买贵价商品时默认会先打开的那个工具。

如果说 V1.2 的目标是“让用户愿意反复回来”，那么 V2 的目标就是“让用户愿意长期把重要商品决策放在这里完成”。
