# Decidely V1.0 功能拆解与开发任务清单

## 1. 文档目的

这份文档用于将 [PRD](/Users/maopengjie/Documents/m-application/docs/PRD.md) 中的 V1.0 范围进一步拆解为可执行的研发任务，方便产品、设计、前端、后端和测试协同推进。

适用范围：

- V1.0 核心链路
- 周级排期
- 任务分工
- 联调和验收

## 2. V1.0 目标

在 V1.0 中，团队需要优先跑通以下核心链路：

`首页 -> 搜索 -> 搜索结果 -> 商品详情 -> 去购买 / 设置提醒`

V1.0 的判断标准不是功能数量，而是用户是否能顺畅完成一次购买前决策。

## 3. 任务优先级定义

- `P0`：上线阻塞项，缺失会导致主链路不成立
- `P1`：重要优化项，不阻塞上线，但会影响体验和可信度
- `P2`：可延后项，本期不进入开发主计划

## 7. 联调清单

- 搜索页筛选参数是否与接口一致
- 搜索结果列表字段是否与设计一致
- 详情页切换 SKU 后所有模块是否联动更新
- 建议卡字段是否完整且可读
- 提醒创建是否绑定当前用户
- 提醒列表状态是否准确
- 用户端导航是否已隐藏后台入口

## 8. 验收清单

### 主链路验收

- 用户可从首页发起搜索
- 搜索结果可点击进入详情
- 详情页可完成去购买或设置提醒
- 提醒创建后可在提醒列表中看到

### 体验验收

- 详情页首屏清晰表达当前建议
- 用户可以理解推荐购买方案与最低价方案差异
- 无数据与报错场景有明确提示

### 权限验收

- 普通用户无法看到后台运营入口
- 提醒仅可查看和删除自己的数据

## 9. 风险与阻塞项

- 建议模块后端字段不足，可能导致前端需临时拼装
- 历史价格数据不足，影响趋势解释可信度
- 风险模块仍偏演示逻辑，可能影响整体信任感
- 若导航拆分范围不清，会拖慢联调和验收
