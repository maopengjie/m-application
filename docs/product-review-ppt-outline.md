# Decidely 产品评审 PPT 大纲

## 1. 封面

- 项目名称：Decidely
- 副标题：智能购物决策与价格监控平台 V1 评审
- 评审日期
- 评审人

## 2. 项目背景

- 高客单商品购买决策成本高
- 用户需要同时判断价格、平台、店铺和优惠
- 现有平台更偏交易，不偏决策
- 机会点：在下单前提供独立决策层

建议一句话：

`我们不是要替代电商平台，而是做用户购买前的一次关键判断。`

## 3. 用户痛点

- 不知道现在买会不会买贵
- 不知道哪个平台更划算
- 最低价不一定最靠谱
- 没到心理价位时缺少持续跟踪机制

可配一页用户决策流程图：

`想买商品 -> 到处比价 -> 看评论和优惠 -> 担心买亏/踩坑 -> 犹豫下单`

## 4. 产品定位

- 一句话定位：买高客单商品前，先看它现在值不值得买
- 核心用户：手机、笔记本、家电等高客单商品购买者
- 核心价值：
  - 判断值不值得买
  - 判断去哪买
  - 判断有没有风险
  - 不买时沉淀为提醒

## 5. 为什么现在做

- 高客单商品天然有“研究决策”需求
- 比价与历史价格已有明确用户心智
- 风险与提醒能进一步构成差异化
- 项目当前已有前后端基础，具备快速验证条件

## 6. V1 产品目标

- 跑通 `搜索 -> 详情 -> 决策 -> 去购买/设置提醒`
- 让详情页成为核心决策页
- 用提醒机制验证留存与回访

非目标：

- 不做全品类平台
- 不做泛 AI 助手
- 不做复杂社区生态

## 7. 用户主链路

主链路：

`首页 -> 搜索 -> 搜索结果 -> 商品详情 -> 去购买 / 设置提醒`

补充链路：

`提醒触发 -> 返回详情 -> 再次决策 -> 去购买`

这一页建议重点强调：

`详情页是核心转化页，提醒页是留存承接页。`

## 8. 核心方案

### 8.1 首页

- 聚焦搜索，不做信息堆砌
- 热门品类和热门降价商品作为辅助入口

### 8.2 搜索结果页

- 通过价格、品牌、平台和标签帮助用户快速筛选
- 核心目标是促成详情点击

### 8.3 商品详情页

- 商品信息
- 决策建议
- 多平台比价
- 历史价格趋势
- 优惠说明
- 风险提示
- 去购买 / 设置提醒

## 9. 决策建议模块

这是最重要的一页，建议单独讲。

模块结构：

- 结论：建议买 / 建议等 / 谨慎购买
- 证据：价格相对历史低点位置
- 风险：店铺或低价风险
- 动作：去购买 / 设置提醒 / 看替代品

示例表达：

- 结论：建议等待
- 证据：当前价比近 60 天低点高 8%
- 风险：当前最低价来自非旗舰店
- 动作：设置 4599 元提醒

## 10. 版本范围

### P0

- 首页搜索入口
- 商品搜索
- 搜索结果列表
- 商品详情页
- 决策建议模块
- 多平台比价
- 历史价格趋势
- 降价提醒

### P1

- 决策解释增强
- 店铺可信度表达
- 优惠到手价解释
- 替代商品推荐
- 收藏或关注
- 埋点与数据看板

### P2

- AI 对话导购
- 个性化推荐
- 深化评论分析

## 11. 产品路线图

### V1.0（第 1-4 周）

- 跑通核心决策链路

### V1.1（第 5-8 周）

- 强化可信度与解释力

### V1.2（第 9-12 周）

- 做提醒回访与留存验证

建议在页尾加一句：

`先做深决策，再做强留存，最后再扩功能。`

## 12. 关键指标

- 搜索用户数
- 搜索到详情点击率
- 详情页去购买点击率
- 详情页提醒创建率
- 提醒触发回访率
- 7 日复访率

这一页要表达的是：

`我们不是只做功能，而是要验证行为变化。`

## 13. 风险与挑战

- 用户不信任建议
- 数据刷新不及时影响判断
- 风险能力真实度不足
- 前后台边界不清
- 提醒有创建但回访不强

## 14. 需要评审确认的事项

- V1 首期聚焦品类是否明确为手机、笔记本、家电
- 是否弱化独立优惠页和风险页
- 用户端与运营端是否拆导航
- 替代商品推荐是否纳入 V1.1
- 提醒触达方式优先级如何定义

## 15. 结论

- Decidely V1 的目标不是做另一个购物站
- 它要成为高客单商品购买前的决策工具
- 核心成败点是：用户是否愿意相信建议并采取行动
- 当前最优先的，不是加更多功能，而是把决策链路做深做顺
