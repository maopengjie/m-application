# Decidely V1.1 需求清单

## 1. 版本目标

V1.1 的目标不是增加大量新功能，而是把 V1.0 从“能用”升级到“可信、好用、可验证”。

V1.1 聚焦三件事：

- 提升决策建议的可信度
- 提升商品详情页的解释力和转化力
- 建立基础数据验证能力

一句话定义：

`让用户不仅能看到建议，而且愿意相信建议。`

## 2. 版本定位

### 2.1 V1.0 到 V1.1 的变化

V1.0 解决的是主链路可用问题：

`首页 -> 搜索 -> 搜索结果 -> 商品详情 -> 去购买 / 设置提醒`

V1.1 解决的是主链路可信问题：

- 用户为什么相信建议
- 用户为什么接受推荐方案
- 用户为什么愿意从详情页采取动作

### 2.2 V1.1 核心结果

- 建议卡从“接口结果”升级为“产品核心模块”
- 详情页从“信息展示页”升级为“决策页面”
- 产品开始具备基础转化验证能力

## 3. 核心需求

### 3.1 决策建议模块增强

#### 目标

把“建议”变成用户真正能理解、能采纳的结论。

#### 需求项

- 建议卡补充固定结构：`结论 + 证据 + 风险 + 动作`
- 展示当前价与历史低点差值
- 展示建议原因文案
- 展示风险说明文案
- 根据建议状态给出不同动作按钮

#### 示例

- 结论：建议等待
- 证据：当前价比近 60 天最低价高 8%
- 风险：当前最低价来自非旗舰店
- 动作：设置提醒 / 查看替代商品

#### 验收标准

- 用户不需要额外理解字段含义
- 建议卡在 5 秒内可读懂
- 不再只显示抽象分数或单一标签

### 3.2 商品详情页解释力增强

#### 目标

让详情页从“信息堆叠”变成“决策页面”。

#### 需求项

- 增加 `30/60/90 天价格位置` 说明
- 增加 `当前价距离历史低点` 文案
- 增加 `推荐方案` 与 `最低价方案` 的区别说明
- 增加 `官方/旗舰/第三方` 的可视化标签
- 优惠模块突出 `到手价`，而不是只列优惠券
- 风险模块突出 `风险原因`，而不是只展示分数

#### 验收标准

- 用户能知道为什么推荐某个平台
- 用户能知道现在买是偏高、正常还是低位
- 用户能快速识别低价是否伴随风险

### 3.3 替代商品推荐

#### 目标

当当前商品“不值得买”时，不让用户链路断掉。

#### 需求项

- 在“建议等待”场景下展示替代商品
- 替代商品推荐规则先做轻量版
- 展示同价位或同品类替代项
- 支持从详情页进入替代商品详情

#### 验收标准

- 当当前商品不推荐购买时，页面仍有后续动作
- 替代商品点击链路顺畅

### 3.4 提醒能力增强

#### 目标

让提醒不只是“创建成功”，而是更有回访价值。

#### 需求项

- 提醒状态展示优化
- 区分 `监控中 / 已触发 / 已失效`
- 在提醒列表中展示价格变化
- 已触发提醒支持直接返回详情页
- 接近目标价的提醒增加提示态

#### 验收标准

- 用户能看懂提醒状态变化
- 提醒页不只是静态列表
- 已触发提醒能带来回访动作

### 3.5 数据埋点与指标看板

#### 目标

让 V1.1 能被验证，而不是只靠感觉判断。

#### 需求项

- 首页搜索发起埋点
- 搜索结果点击详情埋点
- 详情页去购买点击埋点
- 详情页提醒创建埋点
- 提醒触发回访埋点
- 决策卡点击或动作埋点
- 基础看板展示核心转化漏斗

#### 建议优先看的指标

- 搜索到详情点击率
- 详情页去购买点击率
- 详情页提醒创建率
- 提醒触发回访率
- 替代品点击率

#### 验收标准

- 至少能按页面和关键动作查看转化数据
- 能支持 V1.2 排优先级

## 4. 优先级建议

### 4.1 P0

- 决策建议模块增强
- 详情页价格位置解释
- 官方 / 旗舰 / 第三方标签强化
- 到手价解释优化
- 基础埋点

### 4.2 P1

- 替代商品推荐
- 提醒状态增强
- 风险说明增强
- 基础指标看板

### 4.3 P2

- 更复杂的推荐逻辑
- 更复杂的风险算法
- 更复杂的价格预测

## 5. 排期建议

### 第 1 周

- 做 V1.0 数据复盘
- 冻结 V1.1 范围
- 定义建议卡结构和文案规范
- 明确埋点方案

### 第 2 周

- 做决策建议模块改造
- 做详情页解释文案和标签增强
- 做优惠到手价表达优化

### 第 3 周

- 做提醒状态优化
- 做替代商品推荐轻量版
- 做基础埋点接入

### 第 4 周

- 联调、验收、修复
- 输出 V1.1 指标初版
- 决定是否进入 V1.2 留存强化阶段

## 6. 与 V1.0 的关系

V1.1 不应推翻 V1.0 的页面结构，而应在已有主链路上进行强化。

原则如下：

- 不破坏已跑通的核心链路
- 优先增强详情页核心决策能力
- 新能力以提升转化与可信度为优先
- 避免引入与当前定位无关的大功能

## 7. 风险与注意事项

- 若建议卡增强只停留在 UI 层，仍无法真正提升信任感
- 若后端无法提供足够解释字段，前端会出现“样子变好了，逻辑没变”的问题
- 替代商品推荐若质量不高，可能分散用户注意力
- 埋点若定义不清，会导致后续指标无法解释

## 8. 结论

V1.1 的核心任务不是继续扩功能，而是把 V1.0 的主链路变成一条更可信、更有说服力、更可验证的决策链路。

如果说 V1.0 的目标是“让用户能完成一次决策”，那么 V1.1 的目标就是“让用户愿意依据这次决策去行动”。
